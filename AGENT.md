# AGENT Guide

先看文档，再做事。不要跳过顺序，不要直接开始写代码。

推荐阅读顺序：

1. `理解我是谁.md`
2. `新开项目前准备-澄清需求.md`
3. `用户看-这个项目到底要做什么.md`
4. `Agent使用-开始写SPEC文档.md`
5. `新开窗口-根据SPEC完成任务.md`
6. `看仓库-进入research模式.md`

执行规则只有几条：

1. 先理解 Owner 的工作方式。
2. 需求不清时，先澄清，不要直接做方案。
3. 用户确认项目定义后，再写正式 `spec/`。
4. 基于 `spec/` 开始执行时，先失败测试，再最小实现，再验证收口。
5. 文档、代码、测试最终必须一致。

如果当前任务只是 research，就只 research，不改代码。
如果当前任务是开始写 SPEC，就直接写 SPEC，不要多写中间层文档。
如果当前任务是根据 SPEC 完成开发，就按 `新开窗口-根据SPEC完成任务.md` 执行。
