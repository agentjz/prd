# PRD Workspace

这是一个面向 AI 协作开发的轻量文档工作区。这里的核心不是先写代码，而是先把人、需求、项目定义、SPEC 和执行约束写清楚，再让 Agent 按文档推进实现。

阅读顺序很简单：

1. `理解我是谁.md`
2. `新开项目前准备-澄清需求.md`
3. `用户看-这个项目到底要做什么.md`
4. `Agent使用-开始写SPEC文档.md`
5. `新开窗口-根据SPEC完成任务.md`
6. `看仓库-进入research模式.md`

这套文档的作用也很简单：

1. 先让 Agent 理解 Owner 是怎么工作的人。
2. 再通过提问把需求澄清出来。
3. 再写给用户看的项目定义。
4. 再把项目定义转成正式的 `spec/` 文档。
5. 最后基于 `spec/` 开工，并让文档、代码、测试三者保持一致。

如果你是人类读者，就按上面的顺序看。
如果你是 AI Agent，也请先按上面的顺序阅读，不要跳过前面的上下文，尤其不要跳过 `理解我是谁.md`。
